## 系统简介

智能数据分析Agent是一个基于自然语言的数据分析系统，能够：
- 🔍 将自然语言查询转换为SQL
- 📈 执行数据分析（统计、趋势、异常检测）
- 📊 生成交互式可视化图表
- 🧠 记住对话上下文
- 🔄 自动化完成完整数据分析流程

## 数据查询

### 基本查询

1. 进入"🔍 数据查询"页面
2. 在文本框中输入查询内容
3. 点击"🚀 执行查询"按钮
4. 查看SQL预览和查询结果

### 查询示例

| 查询类型 | 示例输入 | 说明 |
|----------|----------|------|
| 统计查询 | "显示销售总额" | 计算总计 |
| 筛选查询 | "查看华东地区的销售" | 条件筛选 |
| 分组查询 | "按地区统计销售额" | 分组聚合 |
| 排序查询 | "显示销售额最高的产品" | 排序展示 |

### 数据导出

查询结果支持以下格式导出：
- **CSV**: 适合Excel打开
- **JSON**: 适合程序处理
- **复制**: 快速复制到剪贴板

## 数据分析

### 分析类型

| 类型 | 说明 | 适用场景 |
|------|------|----------|
| 统计分析 | 计算均值、中位数、标准差等 | 数据概况 |
| 趋势分析 | 分析数据随时间的变化 | 销售趋势 |
| 异常检测 | 发现数据中的异常点 | 异常监控 |
| 对比分析 | 比较不同维度的数据 | 区域对比 |

### 操作步骤

1. 选择数据集（sales/products）
2. 选择分析类型
3. 输入分析目标
4. 配置分析参数（时间范围、粒度等）
5. 点击"🚀 开始分析"

## API调用

### Python API

```python
from main import DataAnalysisAPI

# 创建API实例
api = DataAnalysisAPI(session_id="my_session")

# 执行查询
result = api.query("显示销售总额")

# 获取结果
print(result["response"])  # AI响应
print(result["query_result"])  # 查询结果
```

### 返回值说明

```python
{
    "response": "AI的自然语言响应",
    "query_result": {
        "sql": "生成的SQL语句",
        "data": [[...]],  # 查询数据
        "row_count": 10   # 记录数
    },
    "analysis_result": {...},  # 分析结果
    "visualization_result": {...}  # 可视化结果
}
```

### 流式输出（高级）

```python
# 启用流式输出
result = api.query_stream("显示销售总额")

for chunk in result:
    print(chunk, end="", flush=True)
```

## 技巧和最佳实践

### 查询技巧

1. **明确时间范围**: "最近一个月" 比 "最近" 更准确
2. **使用具体字段**: 指定"销售额"而非"数据"
3. **组合条件**: "华东地区且销售额大于1000"
4. **中文优先**: 系统对中文优化更好

### 分析技巧

1. **选择合适的时间粒度**: 日/周/月根据数据量选择
2. **合理设置Top N**: 太多会影响阅读
3. **先看统计再看趋势**: 建立基本认知
4. **异常检测配合筛选**: 定位具体问题

### 性能优化

1. **避免查询全部数据**: 使用LIMIT限制
2. **适当使用缓存**: 重复查询会更快
3. **清理历史**: 定期清理不需要的记录
4. **监控日志**: 关注响应时间

## 故障排查

### 常见问题

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 查询无结果 | SQL执行错误 | 查看SQL预览检查 |
| API调用失败 | 密钥错误/网络问题 | 检查.env配置 |
| 图表无法生成 | 数据格式错误 | 确认数据完整性 |
| 响应很慢 | API限流/数据量大 | 等待或减少数据量 |

### 日志查看

```bash
# 查看应用日志
tail -f logs/app.log

# 查看错误日志
tail -f logs/error.log

# 搜索错误关键词
grep -i "error" logs/app.log
```
